新建项目

项目环境

  • 安装 Nodejs: https://nodejs.org/en/
  • 安装 babel: npm install babel-cli -g
  • 安装 SVN客户端TortoiseSvn

项目初始化V1.0

  • 安装svn客户端CLI：https://sliksvn.com/download/ (和本机的TortoiseSvn不会冲突)
  • 建立项目文件夹"src"
  • 下载initProject.bat（稳定版）到项目文件夹,并运行initProject.bat
  • 配置项目资源Gruntfile.js (可选）
  • 运行 grunt_coding.bat 初始化项目
  • 运行 grunt_index_server.bat 查看项目开发版运行效果
  • 运行 src/grunt_publish.bat,发布项目到release文件夹
  • 运行 release/grunt_index_server.bat 查看项目发布版运行效果

项目结构V1.0

  • src: 项目源文件
  • release: 项目发布版(发布时自动创建)
  • src/GruntFile.js: 通用库资源配置，其中"watch"参数配置为"true"的资源，由Grunt自动维护
  • src/pageList.html: 项目中所有页面清单
  • src/js/fb_lib/: 通用库资源在项目中的应用
  • src/js/*.js: 项目级js文件
  • src/css/*.css: 项目级css文件
  • src/images/: 项目级图片资源
  • src/fonts/: 项目级图标字体

通用库在项目中的版本控件

  • 开发时，通用库始终使用最新通用库，并实时更新到最新
  • 开发后，前端以测试组测试时的svn版本号（通用库FB_FE_LIB）作为前端版本号
  • 维护时，通用库组件需要改进或升级的，可只针对单个组件升级
  • 资源库的备份与恢复：备份"GruntFile.js"或"_cloudMap.js"，恢复时先恢复备份文件中或从svn上恢复"GruntFile.js"或"_couldMap.js",再运行*_coding.bat即可
